# Я Знаю (iOS, Local Wi-Fi P2P)

Локальная игра для компании друзей (Host/Player) на SwiftUI + Network.framework.

## Логика раунда (текущая)

- Вопросы **не вводятся в приложении**: ведущий задает их устно/из внешнего источника.
- Ведущий открывает раунд кнопкой `Открыть раунд`.
- Игроки нажимают большую кнопку `Ответить`.
- Ведущий видит, кто **первым** нажал кнопку.
- Ведущий вручную оценивает ответ:
  - `Ответ верный (+1)` — игрок получает очко, раунд закрывается.
  - `Ответ неверный` — раунд продолжается, снова ожидается первое нажатие.

## Что реализовано

- Автопоиск ведущего в локальной сети по Bonjour/mDNS.
- Список найденных серверов с именем ведущего у игрока.
- Отдельные настройки ведущего (порт + имя) из `HostLobbyView`.
- Таблица очков у ведущего в реальном времени.
- Устойчивость к разрывам: авто-переподключение клиента и перезапуск сервера.

## Структура проекта

- `quizIOS2/Models/GameModels.swift` — модели и сетевые сообщения.
- `quizIOS2/Network/NetworkManager.swift` — TCP/Bonjour/JSON networking.
- `quizIOS2/ViewModels/AppViewModel.swift` — состояние, раунды, очки, логика Host/Player.
- `quizIOS2/Views/Host/HostLobbyView.swift` — лобби и настройки ведущего.
- `quizIOS2/Views/Host/HostControlView.swift` — управление раундом, первый ответивший, оценка, очки.
- `quizIOS2/Views/Player/GameView.swift` — кнопка `Ответить` и статус раунда для игрока.

## Быстрый запуск (на Mac в Xcode)

1. Открыть `quizIOS2.xcodeproj`.
2. Запустить приложение минимум на 2 устройствах в одной Wi-Fi сети.
3. На одном выбрать роль `Ведущий`, при необходимости настроить порт.
4. На втором выбрать роль `Игрок`, выбрать ведущего из списка и подключиться.
5. Начать игру: ведущий открывает раунд, игроки нажимают `Ответить`.
